Deploy on Render (forms + static pages)
========================================

This site is mostly static HTML/CSS, but forms POST to a small Node server
(server.js) that saves submissions under ./data as JSON lines (for testing).
Render should use a Web Service (this repo includes render.yaml), not a
Static Site only.

Prerequisite: Git repository on GitHub, GitLab, or Bitbucket.

Steps (dashboard)
-----------------
1. Push this project to your host (include package.json, server.js, render.yaml).
2. https://dashboard.render.com → New → Web Service (or Blueprint if you prefer).
3. Connect the repo. Render will suggest settings from render.yaml if you use Blueprint.
4. If creating Web Service manually:
   - Runtime: Node
   - Build command:  npm install
   - Start command:  npm start
   - Instance type: Free (or paid if you prefer always-on).
5. Deploy. Open the .onrender.com URL.

Local testing
-------------
  cd this folder
  npm install
  npm start
  Open http://localhost:3000

Do not open index.html as a file:// URL; form posts need http://localhost
or your Render URL.

Where submissions go
--------------------
If the environment variable DATABASE_URL is set (PostgreSQL connection string),
each POST is stored in the tables from database/schema.postgresql.sql
(contact_leads, care_requests). Run the schema and seed once:

  psql "$DATABASE_URL" -f database/schema.postgresql.sql
  psql "$DATABASE_URL" -f database/seed.postgres.sql

(On Windows PowerShell, use psql with $env:DATABASE_URL.)

Without installing psql (good for Render free + local Windows)
---------------------------------------------------------------
1. In Render: open your **PostgreSQL** → **Connections** → copy **External**
   Database URL (your laptop cannot use the Internal URL).
2. From your project folder on your computer (after git clone), with Node 18+:

     PowerShell:
       $env:DATABASE_URL = "paste-external-url-here"
       npm install
       npm run db:init

   That runs database/run-sql.js (same SSL rules as server.js) and applies
   schema.postgresql.sql then seed.postgres.sql.

3. On your **Web** service, set DATABASE_URL to the **Internal** URL if the web
   service and database are both on Render (recommended); otherwise External.

If DATABASE_URL is not set, each POST is appended to JSONL files for local
testing only:
  data/contact.jsonl          (home page contact section)
  data/contact-leads.jsonl    (About Get In Touch)
  data/care-requests.jsonl   (Care Request Form)

If forms return "We could not save your message"
-------------------------------------------------
1. Web service must have DATABASE_URL pointing at the database where you ran the
   SQL files (Render: link Postgres to the service, or paste Internal/External URL).
2. Run schema + seed (commands above). Log code 42P01 means a table is missing.
3. TLS: non-localhost URLs use SSL by default. Certificate errors: try env
   PGSSL_REJECT_UNAUTHORIZED=false only if your host docs recommend it. For local
   Postgres without TLS, use @localhost in the URL or set DATABASE_SSL=false.

Home page contact form → email (solomon.bagambe1011@gmail.com)
---------------------------------------------------------------
After a successful save (Postgres or JSONL), the server can email a copy.

  **Render FREE web services block outbound SMTP (ports 25, 465, 587).**
  Gmail / nodemailer / SendGrid-SMTP will **time out (ETIMEDOUT)**. Use **HTTPS**
  instead (same port 443 as normal web traffic):

  **SendGrid Web API** (no SMTP):
    SENDGRID_API_KEY=SG....   (app.sendgrid.com → API Keys)
    EMAIL_FROM=verified-sender@yourdomain.com   (Sender Authentication in SendGrid)

  **Resend** (optional alternative):
    RESEND_API_KEY=re_...
    RESEND_FROM=onboarding@resend.dev   (or your verified domain from address)

  EMAIL_TO          optional; defaults to solomon.bagambe1011@gmail.com

  Paid Render instances can use SMTP again if you prefer; order of use in code is:
  SENDGRID_API_KEY → RESEND_API_KEY → SMTP (SMTP_URL or SMTP_HOST).

--- SMTP (local dev, paid Render, or non-Render hosts) ---

  Gmail (same as PHPMailer: host smtp.gmail.com, port 587, TLS, app password):
    SMTP_HOST=smtp.gmail.com
    SMTP_PORT=587
    SMTP_SECURE=tls
    SMTP_USER=your-email@gmail.com
    SMTP_PASS=your-google-app-password
    EMAIL_FROM=your-email@gmail.com
    (Use a Google "App password", not your normal Gmail password.)

  SendGrid SMTP URL (blocked on Render Free — use SENDGRID_API_KEY instead):
    SMTP_URL=smtps://apikey:YOUR_SENDGRID_API_KEY@smtp.sendgrid.net:465
    EMAIL_FROM=verified-sender@yourdomain.com

  Other SMTP hosts: SMTP_SECURE is tls (587) or ssl (465). Optional:
    SMTP_REQUIRE_TLS=false   if STARTTLS causes issues on port 587.

If no mail env is set, the form still saves; only the notification email is skipped.
Deploy logs show: "home email: SendGrid API (HTTPS)" / "Resend..." / "SMTP" / "off...".

Admin dashboard (retrieve database rows)
----------------------------------------
1. Set **ADMIN_PASSWORD** on the Web service (use a long random value). Optional: **ADMIN_USER** (default `admin`).
2. Open **https://YOUR-SERVICE.onrender.com/admin/index.html** (or `/admin` → redirects).
3. When the browser asks for credentials, enter **ADMIN_USER** / **ADMIN_PASSWORD**.
4. Tabs: contact leads, care requests, scheduled visits (Postgres), and JSONL backup files when DB is off.

Notes
-----
- Free web services may spin down after idle; the first request can take ~30s.
- Add spam protection (CAPTCHA) for production if you see abuse.
